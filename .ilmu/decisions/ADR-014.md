# ADR-014: Directory Scanning with walkdir; lint_dir Additive to lint

**Date**: 2026-04-14
**Status**: Accepted

## Context

v0.2.0 adds recursive directory scanning: `<spec>` arg is directory → lint every `.yaml` / `.yml` / `.json` descendant. Architectural questions:

1. `lib::lint()` absorbs this, or new function?
2. What dep handles traversal?
3. One file fails to parse — abort or continue?
4. Where does exit-code logic live?

## Decision

**API shape**:

```rust
// Unchanged — atomic single-file operation.
pub fn lint(spec_path: &Path, ruleset_path: Option<&Path>)
    -> Result<Vec<Violation>, LintError>;

// New — recursive directory scan.
pub fn lint_dir(dir_path: &Path, ruleset_path: Option<&Path>)
    -> Result<Vec<(PathBuf, Result<Vec<Violation>, LintError>)>, LintError>;
```

`lint_dir` returns per-file `Result` inside outer `Ok`. Outer `Err` = directory-level failures (path not a directory, ruleset load fail, I/O error on dir handle). Per-file parse failures → `Err` entries in inner tuple — do **not** abort scan.

**Dependency**: `walkdir = "2"` — MIT/Apache, pure Rust, no transitive C deps, de-facto standard for recursive filesystem traversal. Confirmed musl-safe.

**File selection**: `walkdir` iterates all files; filter by extension (`.yaml`, `.yml`, `.json`, case-insensitive). Symlink behaviour: follow by default (matches `find` and most linters); no descend into cycles (walkdir handles). `.git/` and `node_modules/` not special-cased in v0.2.0 — users exclude by pointing at subdirectory. Add `--ignore <glob>` only if user feedback demands in v0.3.0.

**Error resilience**: file parses but fails OpenAPI version detection → emit stderr warning, record `Err(LintError::InvalidSpec(...))`. `lint_dir` continues. Matches Spectral behaviour.

**Exit code logic stays in `main.rs`**:
- `0` — all files clean
- `1` — at least one file had violations
- `2` — at least one file failed to parse OR directory-level op failed

Lib returns structured data; main decides policy. Keeps `lib` pure, testable, no `std::process::exit` entanglement.

## Consequences

- `main.rs` grows small dispatch: `cli.spec` is directory → call `lint_dir`; else `lint`. Both paths collect into `&[(PathBuf, Vec<Violation>)]` shape reporter expects.
- No async runtime needed — synchronous walkdir traversal sub-second for typical monorepos (<1000 specs). If v0.3.0 benchmarks show bottleneck, `rayon::par_bridge` over walkdir iterator is drop-in parallelisation (see ADR-010 note).
- Symlink-following defensible default but can surprise users with wide-symlinked workspaces. Document in README; revisit if issues arise.
- `Result<Vec<(PathBuf, Result<...>)>, LintError>` return type slightly unusual. `ScanReport` newtype could replace in v0.3.0 if ergonomics need it; tuple shape explicit, easy to pattern-match.
