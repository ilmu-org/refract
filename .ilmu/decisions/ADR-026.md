# ADR-026: v0.5.0 Rule Set, 6 Spectral OAS Parity Rules

**Date**: 2026-04-16
**Status**: Accepted

## Context

v0.4.0 closed JSON Schema validation gap: refract runs `oas3-schema`, `oas2-schema`, `oas3-valid-schema-example`, `oas2-valid-schema-example` via boon (ADR-022), plus cross-file `$ref` resolution (ADR-023). 36 rules shipped.

Audit of Spectral OAS core ruleset vs refract registry found 7 remaining candidates:

| Rule | Class | Notes |
|------|-------|-------|
| `oas2-parameter-description` | structural, OAS 2.x | Mirror of `oas3-parameter-description`. Parameters may be `$ref`, deref-before-compare applies. |
| `oas2-api-schemes` | structural, OAS 2.x | Top-level `schemes` array present, non-empty, values from `{http, https, ws, wss}`. |
| `oas2-anyOf` | structural, OAS 2.x | `anyOf` not valid in OAS 2.x. Walk schemas, flag occurrences at source. |
| `oas2-oneOf` | structural, OAS 2.x | `oneOf` not valid in OAS 2.x. Walk schemas, flag occurrences at source. |
| `oas3-valid-media-example` | boon, OAS 3.x | MediaType `example` / `examples` validated against sibling `schema`. Distinct from `oas3-valid-schema-example` (triggers on schema-shaped nodes only). MediaType holds `example` at MediaType level, not inside schema. |
| `oas2-valid-media-example` | boon, OAS 2.x | OAS 2.x Parameter and Response `examples` map validated against sibling schema. |
| `oas3-unused-component` | graph traversal, OAS 3.x | Detect unreferenced entries in `components.schemas`, `components.parameters`, etc. Needs full document reference graph. Deferred, see ADR-027. |

**Milestone-size reference:**

- v0.3.0: 17 structural rules. Upper edge of reviewable.
- v0.4.0: 4 rules + resolver + boon integration. Two heavy infra pieces.
- v0.5.0 candidates (6 rules after deferring unused-component): reuse existing infra (deref, LintContext, boon, OasVersion gating). No new crate deps. Tight review surface.

**Deref-dependent rule count after v0.5.0:**

Current deref-dependent files: 7 (array_items, oas3_parameter_description, operation_parameters, operation_tag_defined, oas3_valid_schema_example, oas2_valid_schema_example, path_params).

New deref-dependent rules in v0.5.0: 3 (oas2-parameter-description, oas3-valid-media-example, oas2-valid-media-example).

`oas2-anyOf` and `oas2-oneOf` NOT deref-dependent: detect keyword at definition site, not through refs. `$ref` pointing to schema with `anyOf` produces violation on target schema (reporter path), not ref site.

Total after v0.5.0: 10. Exceeds ADR-021's >8 trigger for `Deref'd<'a>` newtype. See ADR-028 for deferral rationale.

## Decision

v0.5.0 ships 6 rules. No new crate deps. No infra refactors.

| Rule | Severity | OAS version | Deref | Boon |
|------|----------|-------------|-------|------|
| oas2-parameter-description | warn | 2.x only | yes | no |
| oas2-api-schemes | warn | 2.x only | no | no |
| oas2-anyOf | error | 2.x only | no | no |
| oas2-oneOf | error | 2.x only | no | no |
| oas3-valid-media-example | error | 3.x only | yes | yes |
| oas2-valid-media-example | error | 2.x only | yes | yes |

**Rule details:**

`oas2-parameter-description`: walk every Parameter Object in `paths.*.parameters` and `paths.*.*.parameters`. Deref `$ref` before inspecting `description`. Violation if `description` missing or empty string. Skip parameters with unresolvable external `$ref` (resolver errors surfaced separately per ADR-023). Source location at parameter node.

`oas2-api-schemes`: read top-level `schemes`. Violation if absent, not array, or empty. Per-entry violation if value not in `{"http", "https", "ws", "wss"}`. No deref; `schemes` is top-level scalar/array.

`oas2-anyOf` / `oas2-oneOf`: walk whole document. On any object containing `anyOf` / `oneOf` key, emit violation ONLY if schema-shaped. Schema-shaped gate: object must contain at least one of `type`, `properties`, `allOf`, `oneOf`, `anyOf`, `items`, `$ref`. Same `has_schema_key` check as `oas3_valid_schema_example`. Without gate, example payloads with literal `anyOf` key produce false positives. Structural objects (MediaType, Parameter, Response) never pass gate. No deref into `$ref` targets — definition-site detection is Spectral invariant.

`oas3-valid-media-example`: walks MediaType, Parameter, and Header objects. All three have `example` / `examples` alongside sibling `schema` at same level. MediaType in `paths.*.<method>.requestBody.content.*` and `paths.*.<method>.responses.<status>.content.*`. Parameter objects in `paths.*.parameters` and `paths.*.<method>.parameters`. Header objects in `paths.*.<method>.responses.<status>.headers.*`. For each node, resolve `schema` if `$ref`, validate `example` / `examples[*].value` against schema via boon, skip `externalValue`. Truncate at 64 violations per call.

`oas2-valid-media-example`: walk every Parameter/Response `examples` map (OAS 2.x). Validate each example value against sibling `schema` via boon with local compiler (pattern from `oas3-valid-schema-example`). Deref schema if `$ref`. Reuse `strip_example_keys` pattern from v0.4.0. Truncate at 64 violations per call. `externalValue` skipped (no HTTP).

MediaType object (for `oas3-valid-media-example`) detected by presence of `schema` key without schema-shape keys (`type`, `properties`, `items`, `allOf`, etc). Distinct from schema walker in `oas3-valid-schema-example` — MediaType's `example` sits alongside `schema`, not inside it. Parameter and Header objects located by walking known document paths, not heuristic detection.

**Version gating**: all 6 rules use `ctx.version` check at start of `check()` per ADR-021 pattern. oas2-* return early on V3_0/V3_1/Unknown. oas3-* return early on V2/Unknown.

**Severity:**

- Structural parity rules match Spectral severities (warn).
- `oas2-anyOf` / `oas2-oneOf` = error: spec forbids these keywords in OAS 2.x.
- Media example validation = error: invalid example means spec contradicts itself (mirrors `oas3-valid-schema-example`).

## Consequences

- Total built-in rules: 42 (36 + 6).
- Zero new crate deps. Binary size approximately flat.
- Deref-dependent count reaches 10. Newtype deferral is explicit ADR-028 override of ADR-021, not ongoing silence.
- `oas3-unused-component` gap documented in ADR-027. README note updated for near-complete Spectral OAS coverage, unused-component as known gap.
- Phase structure (see plan.md): Phase 1 ships 4 OAS 2.x structural rules (no boon, no shared helpers). Phase 2 ships 2 media-example rules (boon, reuses v0.4.0 patterns). GHA Node.js 24 chore (ADR-025) continues as parallel standalone PR if not yet merged.
- Rule trait, `LintContext`, `OasVersion` helper unchanged. No API surface growth.
