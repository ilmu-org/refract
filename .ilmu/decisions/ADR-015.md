# ADR-015: Document-Internal `$ref` Resolution Utility (Prerequisite for path-params)

**Date**: 2026-04-14
**Status**: Accepted

## Context

`path-params` rule must cross-reference path template tokens (e.g. `{petId}` in `/pets/{petId}`) with parameter objects declared in operation or path level. OAS 2.x: params under `paths.{path}.{method}.parameters` and `paths.{path}.parameters`. OAS 3.x: identical structure.

**The trap**: real-world OAS 3 specs overwhelmingly use JSON Pointer refs to shared parameter components:

```yaml
paths:
  /pets/{petId}:
    get:
      parameters:
        - $ref: '#/components/parameters/PetId'
      responses: ...
components:
  parameters:
    PetId:
      name: petId
      in: path
      required: true
```

Without resolving document-internal `$ref`s, `path-params` sees `{"$ref": "..."}` with no `name: petId` — fires false-positive on virtually every well-structured spec. Unacceptable.

Multi-file `$ref` resolution out of scope for v0.2.0 (per plan.md out_of_scope). Document-internal `$ref`s (starting with `#/`) mandatory.

## Decision

Add shared utility in `src/rules/util.rs`:

```rust
/// Resolve a document-internal JSON Pointer reference.
/// Accepts the form `#/components/parameters/PetId` or `#`.
/// Returns None for external refs (any ref not starting with `#/` or equal to `#`),
/// unresolvable pointers, or malformed input.
pub(crate) fn resolve_internal_ref<'a>(
    doc: &'a serde_json::Value,
    ref_str: &str,
) -> Option<&'a serde_json::Value>;

/// If `value` is an object of exactly `{"$ref": "#/..."}`, resolve it.
/// Otherwise return `value` unchanged. Follows chains up to a fixed depth
/// (default: 16) to prevent cycles.
pub(crate) fn deref<'a>(
    doc: &'a serde_json::Value,
    value: &'a serde_json::Value,
) -> &'a serde_json::Value;
```

`resolve_internal_ref` implements RFC 6901 JSON Pointer semantics (`~0` and `~1` escapes). `deref` is primary API rules call.

`path-params` calls `deref` on every parameter before reading `name` and `in`. If `deref` returns object still containing `$ref` (external/unresolvable), rule treats as opaque and skips matching (avoids false positives). No matching param with `in: path` for a path token = violation emitted.

Utility is `pub(crate)` — not public API. Available to future rules (`reference-components`, `no-unresolved-refs`) without forcing crate-boundary decision now.

## Consequences

- `path-params` correctness tractable. Without utility, rule ships broken.
- `deref` returns reference, no mutation — no clone cost on hot path.
- Cycle detection via fixed-depth recursion is pragmatic; OAS specs rarely chain `$ref` more than 2-3 levels.
- External `$ref` handling intentionally permissive: skip, don't error. Spectral follows external refs when possible; v0.2.0 lacks multi-file resolution so skip is only correct choice.
- Future `$ref`-heavy rules inherit utility free. v0.3.0 multi-file `$ref` can evolve or extend this — API shape small and easy to extend.
