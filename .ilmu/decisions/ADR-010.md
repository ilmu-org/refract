# ADR-010: No Async Runtime for v0.1.0

**Date**: 2026-04-09
**Status**: Accepted

## Context

Tool workload: read one file, parse, run rules, write output. Sequential, CPU-bound, single I/O op. Adding Tokio or async-std would:
- Add ~500K to binary size
- Add compile time
- Add no user-visible benefit (linting single spec not parallelisable in v0.1.0)

## Decision

No async runtime. All I/O synchronous (`std::fs::read_to_string`). Rule engine synchronous. `main.rs` plain synchronous function.

## Consequences

- Binary stays small. Compile times stay short.
- If v0.2.0 adds concurrent multi-file linting or remote `$ref` resolution, Tokio added then. Module boundaries async-ready (all functions pure, no global state).
