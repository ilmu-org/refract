# ADR-018: v0.3.0 Rule Set, 17 Structural and Correctness Rules

**Date**: 2026-04-14
**Status**: Accepted

## Context

v0.2.0 shipped 15 rules: operation metadata, info block, tags, path params, markdown safety. Remaining Spectral OAS gap splits into two classes:

1. Structural/correctness rules: `serde_json::Value` traversal + existing internal `$ref` deref from ADR-015. Examples: `array-items` requires `items` on `type: array`; `no-$ref-siblings` enforces OAS spec invariant; tag and param sanity checks.
2. Schema-evaluation rules (`oas3-schema`, `oas2-schema`, `oas3-valid-schema-example`, `oas2-valid-schema-example`): need JSON Schema evaluator, new dependency, new failure mode, significant binary growth (see ADR-019).

Class 1 = visit nodes, eval predicate, emit violation — same as v0.2.0. Class 2 = new dep + schema eval errors + binary bloat. Single milestone covering both = 30+ rule release with dependency risk in same window. Split isolates dep decision, keeps v0.3.0 reviewable.

## Decision

v0.3.0 ships exactly 17 class-1 rules below. No schema-evaluation rules. No cross-file `$ref` resolution (see ADR-020).

| Rule | Severity | Scope notes |
|------|----------|-------------|
| operation-success-response | warn | Every operation has at least one 2xx or 3xx response. Default response counts as non-success fallback, not success. |
| array-items | error | Any schema with `type: array` must declare `items`. Apply after deref. Skip schemas behind unresolvable external `$ref`. |
| no-$ref-siblings | error | Object containing `$ref` must have no other keys. OAS 3.1 relaxes this for some keywords; v0.3.0 enforces strict rule (matches Spectral default). |
| oas3-api-servers | warn | OAS 3.x only. Top-level `servers` array present and non-empty. |
| path-keys-no-trailing-slash | warn | Path keys don't end in `/` except root key `/`. |
| path-not-include-query | warn | Path keys don't contain `?`. |
| path-declarations-must-exist | warn | No empty `{}` placeholders in path templates. |
| operation-tag-defined | warn | Every tag string on an operation appears in global `tags` array. |
| openapi-tags-uniqueness | error | Global tag `name` values unique. |
| tag-description | warn | Every global tag has non-empty `description`. |
| duplicated-entry-in-enum | warn | `enum` arrays contain no duplicate values (deep equality). |
| typed-enum | warn | Each `enum` value matches declared schema `type`. See ADR-021 for type-coercion semantics. |
| oas3-server-trailing-slash | warn | OAS 3.x only. Server `url` doesn't end with `/`. |
| oas3-server-not-example.com | warn | OAS 3.x only. Server `url` host not `example.com`. |
| oas3-parameter-description | warn | OAS 3.x only. Every parameter (after deref) has non-empty `description`. |
| operation-operationId-valid-in-url | warn | `operationId` matches `^[A-Za-z0-9-._~:/?#[\]@!$&'()*+,;=]+$` (RFC 3986 unreserved + sub-delims + path-safe set). |
| operation-parameters | warn | Within one operation, no two parameters share same `(name, in)` pair after deref. Path-level + operation-level params merged before comparison. |

5 rules version-gated (oas3-api-servers, oas3-server-trailing-slash, oas3-server-not-example.com, oas3-parameter-description, no-$ref-siblings on certain OAS 3.1 keywords). ADR-021 covers gating helper.

4 rules consume `$ref` deref (array-items, oas3-parameter-description, operation-parameters, operation-tag-defined when tags referenced indirectly). ADR-021 codifies deref-before-compare invariant.

## Consequences

- v0.3.0 ships 32 total built-in rules (15 from v0.2.0 + 17 here). Strong Spectral-replacement story for structural correctness.
- No new direct dependency. Binary size stays approximately flat.
- Four schema-evaluation rules and cross-file `$ref` = next milestone (ADR-019, ADR-020). Users needing those today must run Spectral fallback; document in README.
- Rule registration follows v0.2.0 module-per-rule pattern. Count grows linear with file count, no compile-time impact.
- 17 rules = upper edge of reviewable milestone. Impl order: structural-only first (path-keys-no-trailing-slash, path-not-include-query, path-declarations-must-exist, openapi-tags-uniqueness, tag-description, oas3-server-trailing-slash, oas3-server-not-example.com, no-$ref-siblings, oas3-api-servers), then deref-dependent (array-items, oas3-parameter-description, operation-parameters, operation-tag-defined), then type-aware (typed-enum, duplicated-entry-in-enum). Sequencing details in plan.md.
