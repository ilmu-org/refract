# ADR-007: Static Binary Compilation Strategy

**Date**: 2026-04-09
**Status**: Accepted

## Context

"Single static binary, no runtime dependencies" = core user promise. Per platform:

- **macOS**: `x86_64-apple-darwin` and `aarch64-apple-darwin` link against `libSystem.dylib` (always present). Acceptable — macOS has no musl target, system dylib not user-installed. Cross-compile with `cargo build --target aarch64-apple-darwin` on x86_64 macOS or GitHub Actions matrix.
- **Linux**: Must be fully static. Use `x86_64-unknown-linux-musl`. Cross-compile with `cross` or musl Docker image in CI.
- **Windows**: `x86_64-pc-windows-msvc` — links MSVC runtime, ships with Windows. Acceptable. Build in GitHub Actions Windows runner.

No C FFI deps in selected crate set (`serde_yaml` uses `unsafe-libyaml`, pure-Rust YAML parser, not C binding). Enables musl builds without C toolchain.

## Decision

- Release targets: `x86_64-unknown-linux-musl`, `aarch64-unknown-linux-musl`, `x86_64-apple-darwin`, `aarch64-apple-darwin`, `x86_64-pc-windows-msvc`.
- CI builds all five targets in GitHub Actions matrix.
- Set in `Cargo.toml`:
  ```toml
  [profile.release]
  opt-level = "z"       # Optimise for binary size
  lto = true            # Link-time optimisation
  codegen-units = 1     # Single codegen unit for maximum LTO
  strip = true          # Strip debug symbols
  ```
- Distribute via GitHub Releases as compressed tarballs (`.tar.gz` Unix, `.zip` Windows). `brew` tap and `cargo install` as secondary distribution in v0.2.0.

## Consequences

- musl builds need musl toolchain in CI. `cross` (Docker-based cross-compilation tool) handles this without host config.
- LTO + strip + opt-level "z" cuts binary size significantly. Expected: ~3-5MB stripped.
- No CGO, no libc, no system-installed runtime on Linux. Differentiating property vs Spectral.
