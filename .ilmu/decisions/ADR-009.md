# ADR-009: Output Formats — Text (Default) and JSON

**Date**: 2026-04-09
**Status**: Accepted

## Context

CI pipelines consume linter output two ways: human-readable text (PR comments, local dev) and machine-readable JSON (downstream tooling, GitHub Actions annotations, dashboards).

## Decision

Two formats, selected by `--format text|json`:

**Text format** (default):
```
spec.yaml:42:5  error  operation-operationId  Operation must have an operationId.
spec.yaml:18:1  warn   info-contact           Info object should have a contact.
```
Format: `{file}:{line}:{col}  {severity}  {rule-id}  {message}`

ANSI color: `error` = red, `warn` = yellow, `info` = blue. Disabled with `--no-color` or when stdout not a TTY.

**JSON format**:
```json
{
  "violations": [
    {
      "rule": "operation-operationId",
      "severity": "error",
      "message": "Operation must have an operationId.",
      "path": "/paths/~1foo/get",
      "file": "spec.yaml",
      "line": 42,
      "col": 5
    }
  ],
  "summary": {
    "errors": 1,
    "warnings": 1,
    "total": 2
  }
}
```

## Consequences

- Text format grep-friendly, compatible with most CI log renderers.
- JSON format enables GitHub Actions problem matchers and downstream dashboards.
- ANSI detection via `std::io::IsTerminal` (stable since Rust 1.70). No extra crate needed.
- `serde_json` (already dependency) handles JSON serialisation.
