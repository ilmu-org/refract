# ADR-012: Flat `line`/`col` Fields on Violation (Not a Location Wrapper)

**Date**: 2026-04-14
**Status**: Accepted
**Supersedes**: none (extends ADR-009 for v0.2.0)

## Context

Two shapes for source position on `Violation`:

```rust
// Option A (flat):
pub struct Violation {
    pub rule_id: String,
    pub message: String,
    pub severity: Severity,
    pub path: String,
    pub line: Option<u32>,
    pub col: Option<u32>,
}

// Option B (wrapper):
pub struct Location {
    pub path: String,
    pub line: Option<u32>,
    pub col: Option<u32>,
}
pub struct Violation {
    pub rule_id: String,
    pub message: String,
    pub severity: Severity,
    pub location: Location,
}
```

Wrapper justified only when: (1) grouped fields share invariants, or (2) type substituted across multiple contexts. Neither applies:

- Three location fields independent — `path` always present, `line`/`col` may be absent (JSON specs, unknown paths).
- No second "thing with a location" in domain.
- SARIF maps directly: `ruleId ← rule_id`, `level ← severity`, `message.text ← message`, `locations[0].logicalLocations[0].fullyQualifiedName ← path`, `locations[0].physicalLocation.region.startLine ← line`, `startColumn ← col`. Wrapper adds no alignment benefit.

## Decision

Adopt Option A. Extend `Violation`:

```rust
#[derive(Debug, Clone, serde::Serialize)]
pub struct Violation {
    pub rule_id: String,
    pub message: String,
    pub severity: Severity,
    pub path: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub line: Option<u32>,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub col: Option<u32>,
}
```

`#[serde(skip_serializing_if = "Option::is_none")]` keeps JSON clean when line/col absent.

## Consequences

- All 8 existing rules compile unchanged — never construct `Violation` with `line`/`col` set. Use `Default` or struct update syntax. Add inherent constructor `Violation::new(rule_id, message, severity, path)` leaving `line`/`col` as `None`. `lib::lint()` fills them after from `PositionIndex`.
- JSON backwards-compatible with v0.1.0 consumers when line/col absent (fields don't appear).
- Text output gains conditional `:line:col` suffix: `{file}:{line}:{col}  ...` when present, `{file}  ...` when absent. Reporter formats prefix accordingly.
- Future multi-file need (e.g. `$ref` resolution across files) → flat-to-wrapper refactor is mechanical. YAGNI wins now.
