# ADR-025: GitHub Actions Node.js 24 Migration

**Date**: 2026-04-15
**Status**: Accepted

## Context

GitHub deprecating Node.js 20 runtime for Actions. Effective June 2026. Node.js 20 actions emit warnings, eventually fail.

**Current action versions and their Node.js runtime:**

| Action | Current version | Node.js runtime | Latest version | Latest runtime |
|--------|----------------|-----------------|----------------|----------------|
| actions/checkout | @v4 | node20 | @v6.0.2 | node24 |
| actions/upload-artifact | @v4 | node20 | @v7.0.1 | node24 |
| actions/download-artifact | @v4 | node20 | @v8.0.1 | node24 |
| softprops/action-gh-release | @v2 | node20 | @v3.0.0 | node24 |

All four actions in release workflow use Node.js 20. CI workflow uses only `actions/checkout@v4`. Both need updates.

**Risk assessment:**

Version bumps can introduce behavioral changes. Known risks:
- `actions/checkout@v6` may change default `fetch-depth` or sparse checkout behavior (check release notes).
- `actions/upload-artifact@v7` and `download-artifact@v8` changed artifact storage format in v4 series; v5+ maintains v4 format but verify upload/download cross-version compatibility within same workflow run.
- `softprops/action-gh-release@v3` is major bump; review release notes for breaking input parameter changes.

**Other actions in workflows:**

`Swatinem/rust-cache@v2` and `dtolnay/rust-toolchain@stable` in `ci.yml` are composite actions, not JavaScript. No Node.js runtime. Not affected by Node.js 20 deprecation. No bump needed.

**Coupling to v0.4.0:**

Infrastructure-only. No Rust changes. No new rules. Orthogonal to v0.4.0 feature work (resolver, boon, schema validation rules).

## Decision

Ship Node.js 24 migration as standalone chore PR against `main`, separate from v0.4.0 build branch. Decoupled so action version bump CI regression can't block v0.4.0. Breakage: revert without touching v0.4.0 branches.

**Target versions:**
- `actions/checkout@v4` -> `actions/checkout@v6` (skipping v5, also node24 but v6 is current stable)
- `actions/upload-artifact@v4` -> `actions/upload-artifact@v7`
- `actions/download-artifact@v4` -> `actions/download-artifact@v8`
- `softprops/action-gh-release@v2` -> `softprops/action-gh-release@v3`

**Testing strategy:** Open chore PR. All CI jobs run on it (including release matrix build if possible via `workflow_dispatch` or tag push). Merge when all checks green. No manual testing beyond CI pass.

**Deadline:** Before June 2026. Not urgent (April 2026) but land before v0.4.0 build branch opens to avoid merge conflicts.

**Responsibility:** `cicd` agent or developer. Not teamlead or architect concern. Assign in v0.4.0 sprint plan as parallel chore task.

## Consequences

- CI workflows use node24 actions across both ci.yml and release.yml.
- No new Node.js 20 deprecation warnings in CI output.
- Chore PR small (~4 line changes), easy to review.
- If `softprops/action-gh-release@v3` has breaking input changes, developer must check release workflow's `with:` block before merging.
- Future action bumps start from node24-based versions.
