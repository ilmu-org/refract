# ADR-027: oas3-unused-component Deferred to v0.6.0

**Date**: 2026-04-16
**Status**: Accepted

## Context

`oas3-unused-component` detects entries in `components.schemas`, `components.parameters`, `components.responses`, `components.requestBodies`, `components.headers`, `components.securitySchemes`, `components.links`, `components.callbacks`, `components.examples` not referenced anywhere in document.

**Scope of reference graph required:**

Reference sites in OAS 3.x go beyond direct `$ref`:

- `$ref: "#/components/..."` — direct pointer.
- `discriminator.mapping` — maps discriminator value to schema ref, string values point at components.
- `security` — arrays of objects keyed by `securitySchemes.<name>`, name reference not `$ref`.
- Operation `security` overrides — same as top-level.
- `Link` object `operationRef` — external operation pointer, similar shape to `$ref` but different semantics.
- Callback objects — nest path items that can `$ref` other components.
- `parameters` may reference components.parameters via `$ref`; schemas inside parameters may reference components.schemas.
- Nested components: schema in `components.schemas.A` may `$ref` another schema in `components.schemas.B`. `B` referenced even if nothing outside components points at it, provided `A` itself is referenced.

Correct reference graph requires transitive closure from roots (paths, webhooks, security) through all reference-bearing fields, then set-difference against defined component keys.

**Spectral precedent:**

Spectral's `oas3-unused-component` has long-running false-positive history: GitHub issues #1340, #1658, #2025 document users disabling rule because `discriminator.mapping`, `Link.operationRef`, and nested `$ref` chains aren't traversed correctly. Refract adopting rule inherits this risk unless graph traversal built deliberately, not pattern-matched.

**v0.5.0 scope pressure:**

v0.5.0 ships 6 structural and boon-based rules (ADR-026). Adding graph-traversal rule with non-trivial correctness risk in same milestone means either:
1. Ship it, accept FP rate similar to Spectral — reputation cost.
2. Build correct graph infra — multi-day refactor, violates ADR-026's "no infrastructure refactors" constraint.

Neither fits v0.5.0's thesis (closing parity gaps with reviewable surface).

## Decision

Defer `oas3-unused-component` to v0.6.0 or later. Do not ship in v0.5.0.

**v0.6.0 prerequisites** (when picked up):

1. Reference graph module (`src/refgraph.rs` or similar) with documented traversal rules covering: `$ref`, `discriminator.mapping`, `security`, `operationRef`, nested refs, callback path items.
2. Test corpus with fixtures for each reference-site class, including FP-adjacent patterns from Spectral's issue tracker (discriminator mappings, circular component references, callbacks).
3. Decision on severity (Spectral uses `warn`; evaluate if `info` more appropriate given FP history).

**v0.6.0 not guaranteed** — if stronger gaps surface (custom rule support, LSP, perf), unused-component slides further.

**Config-detection stub:**

v0.5.0 ships config-detection stub: when `.spectral.yaml` ruleset enables `oas3-unused-component`, refract emits single info-level notice per run: "oas3-unused-component is not yet implemented (see ADR-027). Rule has been skipped." Cost: one config hook in rule dispatcher, no graph traversal logic. Preserves Spectral-config-compatibility story by acknowledging rule rather than silently ignoring. Stub removed when real rule ships.

**Documentation:**

README lists unused-component as known gap with link to this ADR. Users needing it today directed to Spectral for that specific rule, with acknowledgment of Spectral's own FP behaviour.

## Consequences

- v0.5.0 thesis intact: structural and validation parity, no new infrastructure.
- FP risk avoided. Refract keeps "rules fire on real problems" reputation built in v0.2.0-v0.4.0.
- Reference graph module becomes v0.6.0 design task with own ADR. Graph infra also unblocks future rules (unused-tag, unused-parameter, cross-ref correctness).
- Spectral users relying on unused-component must keep Spectral in loop, or disable rule there and accept gap. Acceptable: refract's value is 42 rules it ships cleanly.
- If user files issue requesting unused-component before v0.6.0 scoping, that signal raises priority and tightens graph design requirements (real-world reference patterns).
