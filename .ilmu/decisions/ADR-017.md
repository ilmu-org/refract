# ADR-017: Project Rename — openapi-linter → refract

**Date**: 2026-04-14
**Status**: Accepted

## Context

Project shipped v0.1.0 and v0.2.0 as `openapi-linter`. Name descriptive but generic, indistinguishable from dozens of other linters, no brand identity. Crate never published to crates.io — rename window clean, no downstream breakage.

`refract` chosen: evokes bending a spectrum, deliberate nod to Spectral (Node.js tool this replaces), reads clean as CLI: `refract lint api.yaml`. Short, memorable, domain-appropriate.

## Decision

Rename `openapi-linter` → `refract` at pre-v0.3.0 while reference debt small (two shipped releases, no crates.io publication).

`refract` on crates.io held by 0.0.0 placeholder whose description invites contact. To avoid blocking: **crate name** = `refract-cli`; **binary name** users invoke = `refract`. Explicit `[[bin]]` in `Cargo.toml` decouples them. Owner contactable in parallel — if name transfers before first publish, crate name can change to plain `refract`.

## Migration path for existing users

- GitHub auto-redirects `ilmu-org/openapi-linter` → `ilmu-org/refract` after repo rename.
- Local git remotes need manual update: `git remote set-url origin git@github.com:ilmu-org/refract.git`
- CI configs referencing old binary (`openapi-linter`) must update to `refract`. Binary name change is breaking; v0.2.0 is pre-1.0 so semver permits it.
- `refract-cli` crate name is first-time publish — no existing downstream depends on it.

## Consequences

- Brand identity established before public launch.
- Binary name changes: `openapi-linter` → `refract`. Breaking for existing CI, acceptable at pre-1.0.
- Crate name `refract-cli` differs from binary `refract`. Minor friction for Rust library consumers; `cargo install refract-cli` delivers `refract` binary as expected.
- README carries "Renamed from openapi-linter" breadcrumb until v1.0.0.
